# Java局域网聊天程序开发方案

## 一、项目目标

开发基于Java+Swing的局域网聊天工具，支持多用户在线交互，实现群聊、私聊及消息加密功能.

## 二、核心功能

1. 基础连接：服务器启动后监听指定端口，客户端通过局域网IP与端口发起连接，支持多人同时在线。
2. 聊天模式：群聊消息由服务器广播至所有在线客户端；私聊可从在线用户列表选择目标对象，消息仅双方可见，服务器定向转发。
3. 信息加密：采用AES对称加密方式，客户端发送消息前自动加密，接收后自动解密，服务器仅转发加密后的内容。
4. UI交互：包含连接面板（IP、端口输入框）、在线用户列表、消息显示区、消息输入区，以及群聊/私聊模式切换按钮，操作直观。

## 三、可行性分析
1. 技术门槛低：核心依赖Java Socket（TCP协议）、Swing UI设计、多线程基础，加密功能调用现成工具类，无需深入底层原理。
2. 协作难度小：模块拆分独立，每人负责专属功能，衔接逻辑简单，不易出现跨模块冲突，便于团队配合。
3. 环境易搭建：仅需JDK 8及以上版本，局域网内通过WiFi或网线即可完成测试，无需额外服务器或复杂网络配置。

## 四、开发计划

1. 需求确认与准备：确认消息格式、加密规则等细节，分配具体任务，统一搭建开发环境。
2. 第一阶段并行开发：
   1. 搭建服务器核心框架，完成用户连接与群聊转发基础功能
   2. 封装客户端网络连接、收发消息功能，编写加解密工具类
   3. 用Swing完成客户端界面布局，包含所有必要组件（无需绑定事件）
3. 模块交付与衔接：各模块的可调用雏形完成后，明确对外提供的接口或组件引用；对接UI组件与网络、加密工具，先实现群聊模式下的按钮事件绑定与消息流转。
4. 第二阶段并行开发：
   1. 补充私聊定向转发与在线用户列表维护功能
   2. 完成私聊模式衔接，实现从用户列表选择目标、拼接私聊消息标识等逻辑
   3. 调试加密功能调用，确保兼容性
   4. 优化UI交互细节
5. 整合测试：整合所有模块，解决依赖冲突问题，团队联合测试连接稳定性、群聊/私聊功能有效性、加密解密准确性，排查并记录问题。
6. 优化：修复测试中发现的bug

## 五、具体分工

1. 开发者1（服务器端核心）：搭建服务器框架，维护在线用户列表，先实现群聊消息广播功能，后续补充私聊定向转发逻辑，确保消息分发准确，交付可调用的核心模块。
2. 开发者2（客户端连接+加密）：封装客户端网络连接、收发消息的基础功能，编写简单易用的加解密工具类，提供统一调用接口，配合开发者4完成功能适配，交付网络与加密工具模块。
3. 开发者3（客户端UI设计）：用Swing设计并实现客户端完整界面，包含连接面板、在线用户列表、消息显示区、输入区及模式切换按钮，提供组件引用，按时交付界面布局模块，后续可配合优化交互细节。
4. 开发者4（客户端逻辑衔接）：待开发者1、2、3交付核心模块雏形后，启动衔接工作。绑定UI按钮事件，判断聊天模式并拼接对应消息标识，对接加密工具与网络接口，实现消息“发送-加密-传输-解密-显示”全流程，完成群聊与私聊模式的完整衔接。
5. 开发者5（测试+汇报）：同步跟进各模块开发进度，收集汇报素材；待所有模块开发完成后，牵头进行代码整合与联合测试；最后完成汇报PPT制作、演示环境调试。
